# AGENTS.md — bishou terminal blog

Terminal-themed personal blog SPA. Markdown-driven content, jQuery Terminal UI, GitHub Pages deploy.

## Commands

```sh
npm run dev          # dev server at 127.0.0.1:5173
npm run build        # generate:ascii → tsc -b → vite build
npm run preview      # preview production build
npm run test         # vitest (passWithNoTests — no tests exist yet)
npm run generate:ascii  # avatar PNG → ASCII JSON (requires sharp, called during build)
```

**Build order matters**: `generate:ascii` regenerates `src/assets/avatar-ascii.json` from `src/assets/avatar-source.png`. If the source PNG changes, the JSON will be stale without a full build.

## Architecture

```
src/
├── main.ts              # entry — jQuery terminal instantiation, command dispatch
├── site.config.ts       # nickname/user/host/theme/avatar config
├── styles.css           # all CSS (Catppuccin Mocha theme)
├── content/
│   ├── build.ts         # Vite build-time: load & parse .md, copy images, virtual modules
│   ├── commands.ts      # command parsing, post search/filter by tag
│   ├── file-system.ts   # virtual FS: build file tree, cd/ls/search/completion targets
│   └── types.ts         # Post, KnowledgeBaseEntry, SiteConfig, SiteTheme
├── terminal/
│   ├── completion.ts    # TAB completion logic (commands, cat/cd/tag argument completion)
│   ├── flow.ts          # @chenglou/pretext text flow animation
│   └── render.ts        # HTML renderers: welcome, help, ls output, article, tags, errors
├── avatar/
│   ├── render.ts        # colored ASCII pixel rendering from avatar-ascii.json
│   └── types.ts         # AvatarAsciiArt, AvatarAsciiCell
└── assets/
    ├── ascii-art.txt    # raw ASCII art (fallback, not the colored version)
    ├── avatar-ascii.json # generated by generate:ascii
    └── avatar-source.png # source image → generate:ascii
```

### Content roots

| Directory | Virtual root | Access |
|---|---|---|
| `content/posts/` | `blog/` | `ls ~`, `cd blog`, `cat <title/pinyin/slug>` |
| `content/kb/` | `knowledge base/` (`kb` alias) | `kb`, `cd kb`, `cd linux`, `cat shell` |

### Virtual modules (Vite plugin)

`vite.config.ts` defines a custom plugin that exposes two virtual modules:
- `virtual:posts` → `Post[]` from `content/posts/**/*.md`
- `virtual:knowledge-base` → `KnowledgeBaseEntry[]` from `content/kb/**/*.md`

These are **build-time only** — Markdown is parsed and serialized as JSON embedded in the JS bundle. There is no runtime file reading.

### Image pipeline

1. `prepareContentAssets()` scans the entire repo (excl. `.git/`, `dist/`, `node_modules/`) for images
2. Copies all images into `public/content-assets/` (gitignored, generated)
3. In Markdown: `![[image.png]]` (Obsidian) and `![alt](./path.png)` (standard) are rewritten to `content-assets/...` URLs
4. Asset resolution: resolves relative paths first, then falls back to basename matching with nearest-file tiebreaking

Supported image formats: `avif gif jpeg jpg png svg webp`

## Adding content

### Blog post
Create `content/posts/YYYY-MM-DD-slug.md` with frontmatter:
```yaml
title: "标题"
date: "2026-06-04"
tags: ["tag1"]
categories: ["notes"]
summary: "摘要"
slug: "slug"
draft: false
```
`draft: true` excludes it from the build.

### Knowledge base entry
Create `content/kb/<path>/<file>.md`. Directory structure maps to virtual paths. Frontmatter supports: `title`, `date`, `summary`, `slug`, `draft`.

## Terminal command dispatch

Commands are handled in `src/main.ts` `execute()` switch-case. Adding a command:
1. Add the case in `execute()`
2. Add to `commandNames` in `src/terminal/completion.ts` for TAB completion
3. Optionally add to `renderHelp()` in `src/terminal/render.ts`

## Frontmatter → completion / search

Content is indexed by multiple aliases for Chinese-friendly access:
- English slug (slugified title)
- Original Chinese title
- Pinyin of title (no tones)
- For KB entries: full path pinyin, compact pinyin

## Theme

Catppuccin Mocha colors defined in both CSS custom properties (`:root`) and `site.config.ts` (for terminal prompt colors). If you change colors, update **both**.

## Deployment

Push to `main` → GitHub Actions deploys to GitHub Pages at `https://bishouyou.github.io/bishou-blog/`.
CI runs: `npm ci` → `npm test` → `npm run build` → deploy `dist/` via `actions/upload-pages-artifact`.

`.github/workflows/deploy-pages.yml` — single workflow, build + deploy jobs.

## Dependencies to know

- **jquery.terminal** — the terminal UI. Imported via `jquery.terminal` package. CSS is imported as a raw module.
- **@chenglou/pretext** — text layout engine for flow animations. Used in `src/terminal/flow.ts`. The flow hydration is deferred and batched via `requestAnimationFrame` + `setTimeout(0)`.
- **marked** — Markdown → HTML. Configured with GFM on, async off.
- **gray-matter** — frontmatter parsing.
- **pinyin-pro** — Chinese → pinyin conversion for search/alias generation.
- **sharp** — avatar PNG → ASCII JSON (build script only, dev dependency).
- **fast-glob** — file discovery (build script only, dev dependency).

## Patterns & conventions

- Escape all user/HTML content with `escapeHtml()` / `escapeAttr()` — there are copies in `main.ts`, `render.ts`, `flow.ts`, and `avatar/render.ts`. All do the same thing.
- Path representation: `string[]` arrays (segments). `formatPath()` renders as `~/path/...`.
- `data-command` attributes on clickable elements trigger command execution via delegated click handler.
- The `.codex/` directory contains leftover Codex editor logs. Safe to ignore or delete.
- `dist/` and `public/content-assets/` are generated. Never commit or manually edit.
- `ascii-art-*.zip` and `converted_scan/` are gitignored artifacts.
- `tsconfig.tsbuildinfo` is gitignored incremental build info.

## The init animation sequence

`#terminal.is-starting` class controls the boot animation. Removed after 3200ms. The boot sequence uses CSS `animation-delay`:
- avatar (0ms) → name (1250ms) → about (1680ms) → hint (2320ms) → prompt (2860ms)
